# AWS

## EC2 인스턴스

서버 배포는 리눅스 운영체제를 주로 사용한다.

#### on-promis 온프로미스

별도의 외부 서버를 이용하지 않고, 사내 자체적인 서버 환경을 보유하고 있는 경우



***

#### Docker

컨테이너화 하여 관리하기 위해 사용하는 프로그램, 실행시 자체적으로 프로그램에 필요한 리눅스 환경을 제공한다.\
window 환경에서는 WSL2 를 먼저 설치해서 리눅스 운영체제를 사용가능해야 설치 가능하며, 맥에선 별도 선행 설치 없이 바로 Docker desktop 설치가 가능하다.&#x20;

서버 배포시, 로컬에서는 Docker 를 설치하지 않고 yml 파일만 작성하면 된다. yml 파일 안에 필요한 프론트엔드, 백엔드, DB 의 버전과 설정을 적어서 서버로 보내면, 배포 서버 쪽에서 yml 파일을 열어 그 안에 있는 도커 파일을 실행시켜 서버에서 컨테이너가 만들어진다. \
프론트엔드는 HTTP 포트번호 80 을 작성했지만, SSH 포트 번호인 22는 따로 작성하지 않아도 자동으로 서버를 찾아 들어간다.

Docker-compose 는 여러 컨테이너를 하나의 세트처럼 묶어서 관리하기 위해 사용된다.



***

#### 파일질라

SFTP 를 지원하는 프로그램. \
서버로 파일을 전송하기 위해 사용하며 자체적으로 보안 시스템(Secure)이 들어잇고, \
데이터를 부호화 하여 패킷으로 쪼갠 후 TCP 방식으로 전송한다. 패킷에 필요한 헤더는 개발자가 추가할 필요 없이 자체적으로 붙여서 보내진다.

배포를 위해 빌드된 파일들이 필요하다. 프론트엔드 : dist 폴더 / 백엔드 : jar 파일 / DB : sql 파일

{% hint style="info" icon="hand-point-up" %}
**jar VS war 배포의 차이는 무엇인가요?**

jar 배포는 SpringBoot 프로젝트에서 주로 사용하며, 톰캣이 자체 내장되어 있기 때문에 jar 파일만 실행하면 웹서버까지 함께 실행됩니다. \
반면, war 배포는 외부 WAS(Web Application Server) 에 애플리케이션을 배포할때 사용하는 방식으로 주로 스프링 레거시 프로젝트에서 사용합니다. 톰캣이 자체 내장되어 있지 않기 때문에 별도의 WAS 설치가 필요합니다.
{% endhint %}
