# Javascript

## “use strict” (엄격모드)

> “엄격 모드” 로 실행되어야 함을 알리는 지시어

* 스크립트 시작에 추가되면 전역 범위에 영향 미침.
* 함수 안에 추가 되면 함수 안 (= 로컬) 에서만 영향 미침.
* “use strict” 지시어를 이해하는 컴파일러 에서만 의미 있음

<pre class="language-javascript"><code class="lang-javascript"><strong>&#x3C;script>
</strong><strong>"use strict"
</strong><strong>&#x3C;/script>
</strong><strong>// 깔끔한 코딩에 도움되고,
</strong>// F12(브라우져의 개발자모드)의 console 창에서 에러 확인 가능
</code></pre>

<details>

<summary>선언되지 않은 변수는 사용 불가 (아래경우 에러 발생)</summary>

1. 쓰기 권한 없는 속성에 속성값 쓰기
2. 존재하지 않는 속성 사용
3. 존재하지 않는 변수 사용
4. 존재하지 않는 객체 사용

</details>

## **( )**

소괄호 안에는 인자, 조건 등이 들어간다\
문장 안의 부속품들이 많이 들어간다.

## **{ }**

중괄호\
코드 블럭은 무조건 중괄호 안에 넣는다.\
if 조건문에서 1줄이라면 중괄호는 생략 가능, 1줄 이상이라면 필수

*   예외사항

    ⇒ case 문에는 () {} 를 붙이지 않는다.

소괄호보다 더 범위가 넓음. 문장들을 묶어줄때는 주로 중괄호를 쓴다.

## **\[ ]**

대괄호\
대괄호 안에는 배열, 타입 등이 들어간다.\
<mark style="color:$info;">대괄호는 데이터들과 많이 관련이 있다. 그래서 지금은 아직 잘 안씀</mark>

## ;

세미콜론. 코드가 끝났다는 걸 알리기 위해 코드 마지막에 붙인다.

*   예외사항

    코드블록은 **자체종결성**을 가지고 있기 때문에 그런 경우에는 ;를 붙이지 않는다.

    → if, switch 조건문, 함수 등

    → 하지만 객체선언 {} 뒤에는 ; 를 붙여야 한다. 리터럴이기 때문?
